# Практична робота №4: Розробка REST API для енергетичних даних.  

Розробка REST API для енергетичних даних

**Тема:** Проєктування та розробка REST API для обміну даними енергетичних об'єктів у форматі JSON

**Виконав:** студент групи ТВ-42 Бондарчук Володимир

## Про проєкт

Цей репозиторій містить вихідний код серверного веб-додатка (REST API), розробленого в рамках практичної роботи №4 з дисципліни "Основи веб-програмування". Головна мета — реалізація надійного бекенду для системи моніторингу та управління сонячними електростанціями.

**Варіант 2:** Сонячна електростанція (СЕС).

### Основний функціонал:

* **CRUD операції:** Повноцінна підтримка створення (POST), читання (GET), оновлення (PUT) та видалення (DELETE) записів про сонячні електростанції.

* **Збереження даних:** Інтеграція з базою даних SQLite для надійного зберігання інформації (потужність, радіація, температура тощо) між перезапусками сервера.

* **Специфічні endpoints:** Наявність спеціалізованих маршрутів для отримання поточних даних генерації (`/generation`) та швидкого запису нових показників із датчиків (`/readings`).

* **Фільтрація:** Можливість гнучкого отримання списку СЕС за їх поточним статусом (наприклад, `?status=active`).

* **Валідація та обробка помилок:** Перевірка наявності обов'язкових полів при створенні об'єктів (400 Bad Request) та захист від звернення до неіснуючих ресурсів (404 Not Found).

* **Тестування:** Вбудований скрипт для автоматизованого тестування всіх маршрутів API.

## Технології

У проєкті використані такі технології:

* **Node.js** — серверне середовище виконання JavaScript.
* **Express.js** — мінімалістичний веб-фреймворк для зручної маршрутизації та обробки JSON-запитів.
* **SQLite / sqlite (npm-пакет)** — легковагова реляційна база даних.
* **Fetch API** — нативний інтерфейс для написання тестового скрипта.
* **Postman** — інструмент для ручного тестування та документування API.

## Як запустити

Проєкт є серверним додатком. Для його запуску на локальній машині:

1. Клонуйте або завантажте цей репозиторій на свій комп'ютер.
2. Відкрийте термінал у кореневій папці проєкту.
3. Виконайте команду `npm install` для встановлення всіх необхідних залежностей (Express, SQLite).
4. Запустіть сервер командою `node server.js`. 
   *Сервер почне роботу за адресою `http://localhost:3000`, а файл бази даних створиться автоматично.*
5. Для запуску автоматичних тестів відкрийте нове вікно терміналу та виконайте команду `node test-api.js`.
